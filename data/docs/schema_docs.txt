DATABASE SCHEMA DOCUMENTATION
==============================

TABLE: customers
Purpose: Stores all customer records who have placed at least one order.
Columns:
  - id: Unique integer identifier for each customer. Primary key.
  - name: Full name of the customer (first and last name combined).
  - region: Geographic sales region the customer belongs to. Values: North, South, East, West, Central.
  - segment: Customer classification by type. Values: Consumer, Corporate, Home Office.
  - joined_date: Date the customer was first registered. Format: YYYY-MM-DD.

TABLE: products
Purpose: Product catalog with pricing and current inventory levels.
Columns:
  - id: Unique integer identifier for each product. Primary key.
  - name: Full product name as displayed in the catalog.
  - category: Product classification. Values: Electronics, Furniture, Office Supplies, Clothing, Food.
  - price: Unit selling price in USD. Does not include discounts.
  - stock: Current inventory count. Items with stock below 10 are considered low stock.

TABLE: orders
Purpose: Every purchase transaction made by customers.
Columns:
  - id: Unique order identifier. Primary key.
  - customer_id: Foreign key referencing customers.id. Links each order to its buyer.
  - product_id: Foreign key referencing products.id. Links each order to the purchased item.
  - quantity: Number of units purchased in this order.
  - total: Final amount charged for this order in USD. Calculated as quantity multiplied by product price.
  - order_date: Date the order was placed. Format: YYYY-MM-DD.

TABLE: returns
Purpose: Records orders that were returned by customers after purchase.
Columns:
  - id: Unique return record identifier. Primary key.
  - order_id: Foreign key referencing orders.id. Links the return to the original order.
  - reason: Customer-stated reason for return. Values: Defective, Wrong item, Changed mind, Damaged in transit, Not as described.
  - return_date: Date the return was processed. Format: YYYY-MM-DD.

RELATIONSHIPS:
  - orders.customer_id → customers.id (many orders per customer)
  - orders.product_id → products.id (many orders per product)
  - returns.order_id → orders.id (one return per order maximum)

JOINS:
  - To get customer name with order: JOIN customers ON orders.customer_id = customers.id
  - To get product name with order: JOIN products ON orders.product_id = products.id
  - To find returned orders: JOIN returns ON returns.order_id = orders.id
  - Full order detail: JOIN all three tables using orders as the central table
